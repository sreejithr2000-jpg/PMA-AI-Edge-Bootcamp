User Journey

From conflict to resolved in minutes

The core flow from calendar connection to autonomous scheduling — with AI working at every step.

1
Connect & Onboard
Connect Google, Outlook, or Apple Calendar via OAuth. Set timezone, optionally select persona. Done in 2 minutes — no complex setup.
Pain: Users abandon tools that require long onboarding before any value is shown.
2
Conflict Detected & Resolved
Overlap detected in <30 seconds. Rules fire first, then AI surfaces a ranked suggestion with reasoning. User approves with one tap or overrides.
Pain: Manual conflict resolution is reactive, stressful, and leaves the other party with no message.
3
AI Sends Decline
Draft decline message generated from user's template. Reschedule suggestion attached. User reviews and approves — or edits — before it sends.
Pain: Writing decline messages is tedious and often skipped entirely, leaving no communication.
4
Follow-ups Captured
5 minutes after meeting ends, attendance nudge fires. User confirms, AI suggests follow-up items based on meeting context. Pushed to Notion or Jira automatically.
Pain: Action items from meetings exist only in memory and are lost within hours without a structured capture system.
User Stories

Requirements grounded in real needs

Each story is grounded in the documented pain points of Priya and Arjun, not hypothetical use cases.

As a student
I want conflicts between my class schedule and work shifts resolved automatically
So that I never miss a mandatory lecture because I forgot to decline a shift, without having to manually compare two different calendars.
As an engineer
I want AI to draft and send decline messages for lower-priority meetings
So that I save the 20+ minutes per week I currently spend writing "sorry I can't make it" emails, while still maintaining professional communication.
As a busy professional
I want AI-suggested follow-up action items captured immediately after a meeting ends
So that action items are never lost to memory, and my Jira/Notion backlog stays in sync with what I actually committed to in meetings.
As a returning user
I want the AI to learn my meeting priority patterns over time and eventually resolve conflicts without my input
So that after 90 days of use, I can fully delegate routine scheduling decisions to the app and focus on doing the work, not managing access to it.
As a privacy-conscious user
I want to view, edit, and permanently delete all behavioral data the app has stored about me
So that I stay in control of my personal data and can trust the product with sensitive calendar information without fear of misuse or lock-in.
Features & Demo

Five core capabilities

Every feature maps directly to a documented pain point. Nothing ships in the MVP that doesn't solve a real problem for Priya or Arjun.

Core ⚖️
3-Layer Conflict Resolution
User-defined rules fire first (sub-100ms). AI suggestion with behavioral context fires second. Manual override is always available. System learns from every override to improve future suggestions.
70%+ AI suggestion acceptance target
AI 🧠
Behavioral Memory Engine
Stores attendance patterns, conflict resolution choices, and follow-up completion rates as vector embeddings. AI retrieves the 5 most relevant past decisions as context for every new suggestion. Improves over 30–90 days of use.
85%+ auto-resolve accuracy at Day 90
AI ✉️
AI Decline Messages
Draft generated from user's excuse template. Reschedule suggestion finds the earliest mutually available slot. User reviews and approves with one tap before anything sends. Explicit permission required; never sends autonomously.
80%+ approve-without-editing rate
Core ✅
Follow-up Capture
5 minutes after a meeting ends, attendance is confirmed and AI surfaces 1–3 context-aware action item suggestions based on the meeting title. User selects and pushes directly to Notion, Jira, or native calendar tasks.
94% follow-up completion (validated)
UX 📊
Attendance Tracking
Binary yes/no nudge fires 5 minutes after every meeting end. Data feeds the behavioral memory engine, surfacing patterns like recurring missed meetings. Adaptive — reduces nudges for meetings consistently attended.
>99% prompt delivery rate target
Core 🗓
Unified Calendar Sync
Google Calendar (webhook, <2s), Outlook/Graph API (webhook, <2s), and Apple CalDAV (5-min polling — no webhook support). All events normalized to a common schema. Conflict detection starts within 30 seconds of any calendar change.
<30s sync-to-detection latency

⚡
Rules-first routing cuts AI costs 65%
User-defined rules resolve 60–70% of conflicts in under 100ms with no Claude API call. AI is only invoked when rules don't match, keeping free-tier costs at ~$0.40/user/month while preserving full AI quality when it matters.
↔
Event-driven for real-time reliability
Webhook-first calendar sync (Google, Outlook) with Redis Pub/Sub for WebSocket message routing means conflicts surface in under 2 seconds. Apple CalDAV is polled every 5 minutes — the only provider without webhook support.
🔐
Privacy by design, not by policy
Calendar tokens stored in secure vault (never in DB). Behavioral data AES-256 encrypted at rest. Attendee names pseudonymized before AI prompts. Zero-retention Claude API mode enabled. Full GDPR/CCPA/FERPA data deletion within 24 hours.
